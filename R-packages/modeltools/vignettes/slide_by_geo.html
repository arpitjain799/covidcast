1. Compute sliding window operations on a COVIDcast signal

Sliding

One of the most fundamental tools in the modeltools package is the slide_by_geo() function, which is based on the family of functions provided by the slider package. The slide_by_geo() function allows a user to compute arbitrary sliding window operations on the values in a covidcast_signal() data frame by passing either a function or formula to the slide_fun argument. Many other functions in modeltools, such as pct_change() and estimate_deriv(), use slide_by_geo() as their workhorse.

We’ll use the same USAFacts reporting data in our examples here that we used in the Getting started guide. The code chunk below fetches the data from the Delphi API.

library(covidcast)
library(dplyr)

source <- "usa-facts"
start_day <- "2020-03-01"
end_day <- Sys.Date() - 5
geo_values <- c("ca","fl","ny","pa")

case_rates <- covidcast_signals(data_source = source,
                               signal = c("confirmed_incidence_prop",
                                          "confirmed_7dav_incidence_prop"),
                               start_day = start_day,
                               end_day = end_day,
                               geo_type = "state",
                               geo_values = geo_values) %>%
  aggregate_signals(format = "wide") %>%
  setNames(gsub(paste0("value\\+0:", source, "_"), "", names(.))) %>%
  arrange(desc(geo_value), desc(time_value))

Sliding with a function

We can pass a function to the slide_fun argument in slide_by_geo(). In this case, the passed function must have the following input argument structure:

  1. x : a data frame with the same column names as the data frame passed to slide_by_geo()

  2. Any number of additional named arguments

  3. An ellipsis ... to allow for any number of additional general arguments

Example 1: Computing a 7-day trailing average by sliding a function

In this example we reproduce the calculation of the smoothed daily case rates that is performed by the Delphi API. This can be done by passing the covidcast_signal data frame of raw daily case rates to slide_by_geo() and then passing a 7-day trailing average function to the slide_fun argument, as follows:

library(modeltools)
case_rates <- slide_by_geo(case_rates, slide_fun = function(x, ...) Mean(x$confirmed_incidence_prop), 
                           n = 7, col_name = "trailing_7dav")

Notice that the above slide_by_geo() call returns a data frame with a new column appended that contains the results of the sliding operation specified by slide_fun. By default, this column is named "slide_value", but alternate names can be passed to slide_by_geo(), as follows:

Since this example is meant to reproduce the 7-day trailing average calculation carried out by the Delphi API, the values in the "trailing_7dav" column should match the pre-computed smoothed daily case rates signal stored in the API (to machine precision). To check this, the code chunk below calculates the maximum absolute difference between the two 7-day trailing averages, over all state-date pairs.

case_rates %>%
  mutate(difference = `trailing_7dav` - confirmed_7dav_incidence_prop) %>%
  filter(time_value >= "2020-03-07") %>%
  summarize(max(abs(difference))) %>%
  as.numeric
## [1] 7.142857e-08

Note: The raw daily case rates that we pass to slide_by_geo() begin on March 1, 2020, so the "trailing_7dav" column created by slide_by_geo() doesn’t represent a true 7-day average until March 7, 2020. This is why we only perform the sanity check calculation above on the dates March, 2020, …, Sys.date() - 5. When a trailing window of n days is not available, slide_by_geo() just applies the slide_fun function/formula to whatever data is available. So for June nth, n = 1,...,6, the values in the "trailing_7dav" column are only n-day trailing averages. Whereas the pre-computed smoothed daily case rates signal stored in Delphi’s API is a true 7-day average for all dates March 1, 2020, …, Sys.date() - 5.

Sliding with a formula

Example 2: Computing a 7-day trailing average by sliding a formula

The code chunk below reproduces the 7-day trailing average from Example 1, but passes a function to the slide_fun argument instead of a formula.

case_rates <- slide_by_geo(case_rates, slide_fun = ~ mean(.x$confirmed_incidence_prop),
                           n = 7, col_name = "trailing_7dav")
